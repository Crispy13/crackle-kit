### Analysis

1.  **The Memory Wall (100KB Tests)**
    * Both SIMD versions hit **~33 GiB/s**.
    * This effectively represents the **physical bandwidth limit** of the RAM for a single core on this system. The code has saturated the hardware; no single-threaded optimization can go faster than this because the CPU is processing data faster than RAM can provide it.

2.  **The Latency Wall (150bp Tests)**
    * Performance is capped at **~8 GiB/s**.
    * The SIMD version is only ~25% faster than the Scalar baseline.
    * **Reason:** For such short sequences, the overhead of the function call (stack setup, loading SIMD constants, pointer arithmetic) dominates the execution time, preventing the SIMD ALU from reaching full utilization.
